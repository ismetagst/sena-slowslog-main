# Sena Slow Blog

> A minimal digital space for slow writing and reading — limited posting to encourage **quality, not quantity**.

Live: [senaslowblog.org](https://senaslowblog.org)

---

## Philosophy

Most platforms reward speed and volume. Sena rewards patience. Writers can only publish on a cooldown (6 days for standard, 3 days for Inner Circle), readers see fewer but deeper pieces, and the entire interface is intentionally quiet — Georgia serif, warm off-white, kaomoji instead of emoji.

This is not a Medium clone. It is a slow space.

---

## Features

### Public
- Landing page with curated published stories
- Read mode with normalized typography (no inline styles, paragraph-based)
- Global search (`Cmd/Ctrl + K`) for articles and users
- Public profile pages at `/@username`
- Footer pages (About, Roadmap, etc.) managed via admin

### Members (Writers)
- Personal Key login (no Google OAuth — see `docs/DECISIONS.md`)
- Waitlist-gated signup
- Distraction-free editor with image upload + client-side compression
- Editor toolbar: bold, italic, underline, strikethrough, bullet list, **blockquote (Medium-style pull quote)**, link, kaomoji, image
- Drafts, soft-delete trash, restore
- Bookmarks
- **Whisper** — private audio notes organized in folders (not part of the public feed)
- Personal analytics (views, devices, growth)
- Achievements (Output, Reach, Special) — DB-managed dynamic badges
- Notifications (Respectful Greetings, Achievements, View milestones every 100)
- Publish cooldown (6 days)

### Inner Circle (Premium)
- Cooldown reduced to 3 days
- "Only My Circle" story visibility
- Trash recovery
- PDF export of own stories
- Voucher-based checkout flow with admin approval

### Admin
- Maintenance mode toggle
- Confidential registration data
- Achievement badges manager
- Footer pages manager
- Payment methods manager
- Voucher manager
- Homepage popup manager
- Toolbar settings

## Conventions

- **Design system**: Georgia serif everywhere. Warm off-white background. Semantic tokens only — never raw colors in components.
- **Kaomoji only**: `(◕ᴗ◕✿)` `(´｡• ᵕ •｡`)` — no emoji in UI copy.
- **Auth**: Personal Key (a passphrase generated on signup). No Google OAuth, no magic links.
- **Destructive actions**: Always wrapped in `AlertDialog` with text confirmation rituals for high-stakes deletes.
- **Editor**: Reading mode strips inline styles and converts `<br>` chains into `<p>` blocks. `<blockquote>` is preserved & styled as a Medium-style pull quote.
- **Roles**: Stored in a separate `user_roles` table — never on `profiles`. Checked via `has_role()` security-definer function. Role badges are **only visible to authenticated users** (anonymous visitors never see them, and there is no `"writer"` fallback).
- **Cooldown**: Enforced server-side, surfaced in UI via `usePublishCooldown`.
- **Notifications**: Generated by Postgres triggers, not by client code.
- **Prefetch**: `StoryCard` prefetches the next story (metadata + RPC content) on hover/intersect so navigation feels instant; `StoryDetail` shows a consistent skeleton during the transition.

## Architecture Notes

- **RLS everywhere.** Every table has Row-Level Security. Anonymous users can only read public stories. Writers can only mutate their own data. Admins use `has_role()`.
- **Roles in a separate table** to prevent privilege escalation via profile updates.
- **Triggers for derived state**: notifications, voucher usage counts, story view milestones, and achievement awards are all DB-driven — the client never decides who gets what.
- **Soft delete**: Stories use `deleted_at`. Hard delete is admin-only.
- **Realtime**: Used for notifications and (where applicable) high-fives / Respectful Greetings.

For a deeper read of the current code health and refactoring priorities, see [`ARCHITECTURE.md`](./ARCHITECTURE.md).
